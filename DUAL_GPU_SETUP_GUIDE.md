# 🚀 FramePack 雙 GPU 設置指南

## 📋 功能概述

FramePack 現在支援雙 GPU 並行處理，通過共享隊列系統實現真正的負載均衡：

- ✅ **獨立 GPU 服務**：每個 GPU 運行獨立的服務實例
- ✅ **共享隊列系統**：所有服務共享同一個處理隊列
- ✅ **自動負載均衡**：任務自動分配給空閒的 GPU
- ✅ **無縫用戶體驗**：用戶可以訪問任一服務上傳圖片

## 🎯 解決的問題

**之前的問題：**
- 排程系統只能啟動一個作業
- PORT 7860 處理作業時，PORT 7861 的隊列是空的
- 無法充分利用雙 GPU 的處理能力

**現在的解決方案：**
- 兩個服務共享同一個隊列
- 上傳到任一服務的圖片都進入共享隊列
- 兩個 GPU 自動從隊列中獲取任務並行處理

## 🛠️ 技術架構

```
用戶上傳 → 任一服務 (7860/7861) → 共享隊列 → 自動分配 → GPU 0/1 並行處理
```

### 核心組件

1. **SharedQueueManager** - 基於文件系統的共享隊列管理
2. **ImageProcessingQueue** - 支援多服務實例的隊列管理器
3. **獨立啟動腳本** - 為每個 GPU 提供專用啟動腳本

## 🚀 使用方法

### 1. 啟動雙 GPU 服務

```bash
# 方法一：使用雙 GPU 啟動腳本（推薦）
./start_dual_gpu.sh

# 方法二：手動啟動各個服務
./start_gpu0.sh  # 啟動 GPU 0 服務 (端口 7860)
./start_gpu1.sh  # 啟動 GPU 1 服務 (端口 7861)
```

### 2. 檢查服務狀態

```bash

```

### 3. 訪問服務

- **GPU 1 服務**: http://211.20.19.206:7861
- **登錄信息**: 用戶名 `admin`，密碼 `123456`

### 4. 停止服務

```bash

```

## 📊 工作流程

### 圖片上傳和處理

1. **用戶上傳圖片**
   - 可以訪問任一服務 (7860 或 7861)
   - 上傳圖片並設置參數

2. **添加到共享隊列**
   - 圖片和參數被添加到共享隊列
   - 隊列狀態在兩個服務間實時同步

3. **自動任務分配**
   - GPU 0 和 GPU 1 自動從隊列獲取任務
   - 不同的任務分配給不同的 GPU
   - 避免重複處理

4. **並行處理**
   - 兩個 GPU 同時處理不同的任務
   - 處理狀態實時更新
   - 完成後自動標記為已完成

### 隊列管理

- **實時同步**：所有服務看到相同的隊列狀態
- **狀態追蹤**：每個任務的狀態（等待/處理中/已完成/失敗）
- **GPU 標識**：顯示哪個 GPU 正在處理特定任務
- **持久化**：隊列數據保存到文件系統，服務重啟後保持

| GPU 0 | 0 | 7860 | CUDA_VISIBLE_DEVICES=0 |

- **隊列目錄**: `./queue_data/`
- **隊列文件**: `./queue_data/queue.json`
- **圖片暫存**: `./queue_data/images/`

## 🔍 故障排除

### 常見問題

**1. 服務啟動失敗**
```bash
# 檢查 Python 環境
/home/jake/.virtualenvs/FramePack/bin/python --version

# 檢查依賴
/home/jake/.virtualenvs/FramePack/bin/python -c "import torch; print(torch.cuda.is_available())"
```

**2. 端口被佔用**
```bash
# 查看佔用進程
lsof -i :7860
lsof -i :7861

# 強制停止
./stop_dual_gpu.sh
```

**3. 隊列同步問題**
```bash
# 清理隊列數據
./stop_dual_gpu.sh clean

# 重啟服務
./start_dual_gpu.sh
```

**4. GPU 記憶體不足**
```bash
# 檢查 GPU 記憶體
nvidia-smi

# 重啟服務釋放記憶體
./stop_dual_gpu.sh && ./start_dual_gpu.sh
```
