# 自動清理功能說明

## 概述

FramePack 現在支持自動清理功能，在圖生影過程中會自動刪除同一任務的較短視頻文件，只保留最新生成的最長版本。這個功能可以顯著節省存儲空間，避免產生大量中間文件。

## 功能特點

### 🎯 智能清理
- **自動觸發**：每次保存新的更長視頻時自動執行
- **精確識別**：只刪除同一任務（相同 job_id）的較短視頻
- **安全保護**：不會影響其他任務的視頻文件

### 💾 空間節省
- **顯著效果**：可節省 60-70% 的存儲空間
- **即時清理**：無需手動操作，實時清理中間文件
- **透明過程**：清理過程在控制台顯示詳細信息

### 🛡️ 安全可靠
- **容錯設計**：清理失敗不會影響視頻生成主流程
- **精確匹配**：基於文件名格式精確識別相關文件
- **錯誤處理**：包含完整的異常處理機制

## 工作原理

### 文件命名格式
視頻文件按以下格式命名：
```
{job_id}_{frame_count}.mp4
```

例如：
- `20250107_123456_48.mp4` - 48幀視頻（約2秒）
- `20250107_123456_96.mp4` - 96幀視頻（約4秒）
- `20250107_123456_144.mp4` - 144幀視頻（約6秒）

### 清理邏輯
1. **保存新視頻**：當生成更長的視頻時
2. **掃描文件**：查找輸出目錄中的所有 MP4 文件
3. **識別相關**：找出相同 job_id 的視頻文件
4. **比較幀數**：識別幀數較少的文件
5. **執行清理**：刪除較短的視頻文件
6. **記錄日誌**：在控制台顯示清理結果

## 使用示例

### 典型的圖生影過程

假設用戶要求生成 10 秒的視頻：

```
⏱️  第1步：生成 2 秒視頻 (48幀)
📁 文件：demo_20250107_123456_48.mp4 (3.2MB)

⏱️  第2步：擴展到 4 秒視頻 (96幀)
📁 新文件：demo_20250107_123456_96.mp4 (6.4MB)
🧹 自動清理：刪除 demo_20250107_123456_48.mp4

⏱️  第3步：擴展到 6 秒視頻 (144幀)
📁 新文件：demo_20250107_123456_144.mp4 (9.6MB)
🧹 自動清理：刪除 demo_20250107_123456_96.mp4

⏱️  最終：完成 10 秒視頻 (240幀)
📁 最終文件：demo_20250107_123456_240.mp4 (16.0MB)
🧹 自動清理：刪除 demo_20250107_123456_144.mp4
```

### 空間節省效果

- **無自動清理**：5個文件，總大小 47.2MB
- **有自動清理**：1個文件，總大小 16.0MB
- **節省空間**：31.2MB (66% 節省)

## 控制台輸出

清理過程會在控制台顯示詳細信息：

```
✅ 自動清理舊視頻: demo_20250107_123456_48.mp4 (3.2MB, 48 幀)
✅ 自動清理舊視頻: demo_20250107_123456_96.mp4 (6.4MB, 96 幀)
```

如果清理失敗，會顯示警告信息：
```
⚠️ 清理舊視頻失敗: demo_20250107_123456_48.mp4 - Permission denied
```

## 配置選項

目前自動清理功能是默認啟用的，無需額外配置。如果需要禁用此功能，可以：

1. **臨時禁用**：修改 `_save_video` 方法，註釋掉清理調用
2. **永久禁用**：在子類中重寫 `_cleanup_old_videos` 方法為空實現

## 注意事項

### 安全性
- 只刪除相同 job_id 的文件
- 只刪除幀數較少的文件
- 包含完整的錯誤處理

### 性能
- 清理操作在後台執行
- 不會顯著影響視頻生成性能
- 文件掃描範圍限制在輸出目錄

### 存儲
- 顯著減少存儲空間使用
- 避免產生大量中間文件
- 保持輸出目錄整潔

## 故障排除

### 常見問題

**Q: 清理功能沒有工作？**
A: 檢查控制台輸出，確認文件名格式正確，檢查文件權限。

**Q: 錯誤刪除了其他任務的文件？**
A: 這不應該發生，功能設計為只刪除相同 job_id 的文件。如果發生，請檢查文件名格式。

**Q: 清理失敗會影響視頻生成嗎？**
A: 不會，清理失敗只會在控制台顯示警告，不會中斷主要流程。
