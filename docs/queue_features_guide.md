# 🎬 FramePack 隊列功能使用指南

## 📋 功能概述

新的隊列功能解決了以下問題：
- ✅ **自動重置上傳組件**：生成完成後無需刷新頁面即可上傳新圖片
- ✅ **批量上傳支持**：一次可以上傳多張圖片進行批量處理
- ✅ **隊列管理系統**：實時查看和管理待處理的圖片隊列
- ✅ **無縫用戶體驗**：所有操作都不需要刷新頁面

## 🚀 新增功能

### 1. 圖片上傳改進
- **單張上傳模式**：傳統的一次上傳一張圖片
- **批量上傳模式**：一次選擇多張圖片文件
- **自動清理**：添加到隊列後自動清空上傳組件

### 2. 隊列管理系統
- **實時狀態顯示**：顯示等待、處理中、已完成的項目數量
- **隊列項目列表**：詳細顯示每個項目的信息
- **隊列控制**：刷新、清理已完成、清空隊列等操作

### 3. 處理流程優化
- **隊列處理**：按順序處理隊列中的項目
- **狀態追蹤**：每個項目的處理狀態實時更新
- **錯誤處理**：失敗項目會被標記並可以重新處理

## 🎯 使用方法

### 基本使用流程

1. **選擇上傳模式**
   ```
   單張上傳 ← → 批量上傳
   ```

2. **上傳圖片**
   - 單張模式：直接拖拽或點擊上傳一張圖片
   - 批量模式：選擇多張圖片文件

3. **設置參數**
   - 輸入提示詞
   - 調整生成參數（解析度、步數等）

4. **添加到隊列**
   - 點擊「添加到隊列」按鈕
   - 圖片和設置會被添加到處理隊列
   - 上傳組件自動清空，可以立即上傳下一批圖片

5. **開始處理**
   - 點擊「開始處理隊列」按鈕
   - 系統會按順序處理隊列中的所有項目

### 隊列管理操作

- **刷新隊列**：更新隊列狀態和項目列表
- **清理已完成**：移除已完成和失敗的項目
- **清空隊列**：清除所有等待中的項目（不影響正在處理的項目）

   - 事件處理優化

3. **BaseApp** 擴展 (`core/base_app.py`)
   - 隊列處理邏輯
   - 批量處理支持
   - 狀態管理

### 數據流程

```
圖片上傳 → 添加到隊列 → 隊列處理 → 生成視頻 → 狀態更新
    ↓           ↓           ↓           ↓           ↓
  清空組件   → 更新列表   → 狀態追蹤   → 保存文件   → 完成標記
```
