# FramePack 運行模式說明

## 問題描述

在遠端開發環境中，當您在 Mac 上使用 `config.env` 配置遠端連接到伺服器時，如果這個配置文件同步到遠端伺服器，直接在遠端伺服器上執行 `start.sh` 會嘗試遠端連接自己，造成問題。

## 解決方案

我們在 `start.sh` 中添加了智能運行模式檢測功能，支持三種運行模式：

### 1. 自動檢測模式 (auto) - 推薦

**使用方法：**
```bash
./start.sh [command]
```

**檢測邏輯：**
- 檢查當前用戶是否與配置的 `REMOTE_USER` 匹配
- 檢查當前目錄是否與配置的 `REMOTE_PROJECT_DIR` 匹配  
- 檢查遠端 Python 環境是否存在
- 如果以上條件都滿足，自動切換到本地模式
- 否則根據配置文件中的 `ENABLE_REMOTE` 設定決定模式

### 2. 強制本地模式 (local)

**使用方法：**
```bash
./start.sh --local [command]
```

**適用場景：**
- 直接在遠端伺服器上執行
- 禁用所有遠端連接功能
- 使用本地 Python 環境和 GPU

### 3. 強制遠端模式 (remote)

**使用方法：**
```bash
./start.sh --remote [command]
```

**適用場景：**
- 從本機遠端連接到伺服器
- 強制使用遠端連接，即使自動檢測建議使用本地模式

## 配置文件設定

在 `config.env` 中添加了新的配置選項：

```bash
# 運行模式 (auto, local, remote)
# auto: 自動檢測 (推薦)
# local: 強制本地模式 (適用於在遠端伺服器上直接執行)
# remote: 強制遠端模式 (適用於從本機遠端連接)
RUN_MODE=auto
```

## 使用示例

### 在 Mac 上遠端開發
```bash
# 自動檢測 - 會使用遠端模式
./start.sh start

# 或明確指定遠端模式
./start.sh --remote start
```

### 在遠端伺服器上直接執行
```bash
# 自動檢測 - 會自動切換到本地模式
./start.sh start

# 或明確指定本地模式
./start.sh --local start
```

## 優先級

運行模式的優先級順序：
1. 命令行參數 (`--local` 或 `--remote`)
2. 配置文件中的 `RUN_MODE` 設定
3. 自動檢測結果
4. 配置文件中的 `ENABLE_REMOTE` 設定

## 注意事項

1. **命令行參數優先級最高**：`--local` 和 `--remote` 參數會覆蓋配置文件和自動檢測結果
2. **自動檢測是智能的**：基於多個條件判斷，通常能正確識別運行環境
3. **配置文件兼容性**：舊的配置文件仍然兼容，只需添加 `RUN_MODE=auto` 即可使用新功能
4. **調試信息**：執行時會顯示詳細的檢測信息，方便排查問題

## 故障排除

如果自動檢測結果不符合預期：

1. **檢查配置文件**：確認 `REMOTE_USER`、`REMOTE_PROJECT_DIR`、`REMOTE_PYTHON` 設定正確
2. **使用命令行參數**：直接使用 `--local` 或 `--remote` 強制指定模式
3. **查看檢測信息**：根據顯示的檢測信息判斷為什麼自動檢測結果不符合預期
4. **手動設定**：在配置文件中設定 `RUN_MODE=local` 或 `RUN_MODE=remote`
