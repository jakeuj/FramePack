## 使用方法

### 1. 正常停止處理

當你想停止當前的處理時：

1. 點擊 **"停止處理"** 按鈕
2. 系統會自動：
   - 停止當前正在處理的任務
   - 清理所有後台線程
   - 將處理中的項目重置為等待狀態

### 2. 清空隊列

當你想清空所有隊列項目時：

1. 點擊 **"清空隊列"** 按鈕
2. 系統會自動：
   - 如果正在處理，先停止處理
   - 清空所有等待中的項目
   - 重置處理中的項目狀態

### 3. 程式重啟

當你重啟 FramePack 程式時：

1. 程式會自動檢查隊列狀態
2. 將任何卡在 "處理中" 的項目重置為 "等待" 狀態
3. 你可以繼續處理這些項目

## 狀態說明

### 隊列項目狀態

- **等待 (waiting)**：項目已添加到隊列，等待處理
- **處理中 (processing)**：項目正在被處理
- **已完成 (completed)**：項目處理成功
- **失敗 (failed)**：項目處理失敗

### 隊列操作

- **添加到隊列**：將新項目添加到隊列末尾
- **開始處理隊列**：按順序處理隊列中的項目
- **停止處理**：停止當前處理，重置狀態
- **清空隊列**：移除所有等待中的項目
- **清理已完成**：移除已完成和失敗的項目

## 故障排除

### 如果隊列仍然卡住

1. 點擊 **"停止處理"** 按鈕
2. 等待 2-3 秒讓系統完成清理
3. 點擊 **"清空隊列"** 按鈕
4. 重新添加項目到隊列

### 如果程式無響應

1. 關閉程式
2. 重新啟動 FramePack
3. 系統會自動重置隊列狀態
4. 檢查隊列狀態，繼續處理

### 檢查隊列狀態

隊列狀態顯示格式：
```
📋 隊列狀態: 等待 X, 處理中 Y, 已完成 Z
```

正常情況下：
- 處理中 (Y) 應該是 0 或 1
- 如果處理中 > 1，說明可能有問題
